# Пользователи системы

**Автор:** (имя второго члена команды)

## 1. Роли пользователей

| Роль | Описание |
|:---|:---|
| **Гость** | Человек, который хочет забронировать номер. Может быть сотрудником другого вуза, приезжим преподавателем, студентом на конференции |
| **Администратор гостиницы** | Сотрудник, управляющий номерами и бронированиями |
| **Неавторизованный посетитель** | Любой человек, зашедший на сайт. Может только смотреть каталог |

## 2. Матрица прав доступа

| Действие | Гость | Администратор | Неавторизованный |
|:---|:---:|:---:|:---:|
| Просмотр каталога номеров | ✅ | ✅ | ✅ |
| Фильтрация по датам | ✅ | ✅ | ✅ |
| Бронирование номера | ✅ | ✅ | ❌ |
| Просмотр истории своих броней | ✅ | ✅ | ❌ |
| Просмотр всех бронирований | ❌ | ✅ | ❌ |
| Подтверждение брони | ❌ | ✅ | ❌ |
| Отмена брони | ❌ | ✅ | ❌ |
| Редактирование информации о номерах | ❌ | ✅ | ❌ |

## 3. Пользовательские истории (User Stories)

### Гость:
- Как гость, я хочу видеть список свободных номеров на нужные даты, чтобы выбрать подходящий вариант.
- Как гость, я хочу видеть фотографии номеров до бронирования, чтобы оценить комфорт.
- Как гость, я хочу зарегистрироваться через email, чтобы сохранять историю броней.
- Как гость, я хочу получать подтверждение брони на email.

### Администратор:
- Как администратор, я хочу видеть все бронирования на сегодня, чтобы подготовить номера.
- Как администратор, я хочу подтверждать или отклонять бронирования.
- Как администратор, я хочу менять статус номера (доступен / занят / ремонт).

## 4. Требования к интерфейсу

- На главной странице: поиск по дате заезда, дате выезда, количеству гостей
- Карточка номера содержит: фото, название, цену, вместимость, кнопку «Забронировать»
- Цветовая схема соответствует брендбуку РГПУ им. А. И. Герцена
- Логотип университета в верхней части сайта
- Форма бронирования: поля «Имя», «Email», «Телефон», «Дата заезда», «Дата выезда»
